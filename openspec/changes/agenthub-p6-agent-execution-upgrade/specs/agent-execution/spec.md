## ADDED Requirements
### Requirement: P6 执行能力定位
系统 MUST 在保留 P4/P5 本地单用户工作区基线以及现有 `CodexAdapter`、`ClaudeCodeAdapter` 和 `ScriptedMockAdapter` 的同时，升级了 AgentHub 的实际代理执行能力。

#### Scenario: P6 范围已审查
- **WHEN** 维护者审查 P6 变更
- **THEN** 该变更将 P6 描述为代理执行能力升级
- **并且** 保留了现有的差异对比、审查、预览、制品卡片和模拟部署流程
- **并且** 未声称具备任意 SaaS 生成、生产环境部署、多用户即时通讯、提供商市场、Docker 沙箱或 PR 创建功能。

### Requirement: 消息路由与直接代理分配
系统 MUST 默认将未明确提及角色的消息路由至编排器/管理器，并 MUST 将明确提及角色的消息视为更高优先级的分配指令。系统 MUST 通过现有 TaskRun 路径自动启动由编排器创建的安全演示目标编码任务。

#### Scenario: 用户发送的消息未明确提及角色
- **WHEN** 用户发送一条不含 `@orchestrator`、`@frontend`、`@backend`、`@qa` 或 `@review` 的普通消息
- **THEN** 系统将消息路由至编排器/管理器
- **且** 编排器能够以对话方式回复、创建角色任务、生成合约或任务图、请求澄清，或如实拒绝不支持的请求
- **且** 系统不会静默执行其他角色的适配器运行

#### Scenario: 编排器创建一个安全的演示编码任务
- **WHEN** 编排器根据普通消息或 `@orchestrator` 消息创建一个安全的演示目标前端编码任务
- **THEN** 系统通过现有的 TaskRun 执行路径自动启动一个 TaskRun
- **并且** TaskRun 使用已配置的现有代码适配器
- **并且** 任务指令保留用户的原始请求和安全的演示目标边界。

#### Scenario: 用户选择了一个未明确提及角色的制品
- **WHEN** 用户发送了一条包含所选制品上下文的后续消息，但未明确提及角色
- **THEN** 所选制品已包含在编排器上下文中
- **且** 所选制品默认不绕过编排器
- **且** 编排器决定下一步任务、澄清或拒绝。

#### Scenario: 用户明确提及编码代理
- **WHEN** 用户发送一条受支持的直接提及，例如 `@frontend 更新演示应用首页文案` or `@backend 为演示 API 添加联系人端点`
- **THEN** 系统创建一个可执行的任务，并分配给被提及的角色
- **并且** 该任务保留用户的原始请求
- **并且** 该任务可通过现有的 TaskRun 路径启动
- **并且** 该任务被限定在该角色的安全目标目录内。

#### Scenario: 用户明确提及编排器
- **WHEN** 用户发送一条包含 `@orchestrator` 的消息
- **THEN** 系统将消息路由至编排器/管理器
- **并且** 编排器应用与默认路由相同的决策策略。

#### Scenario: 用户直接提及不支持的职责请求
- **WHEN** 用户发送了一个无法安全界定范围的直接提及请求
- **THEN** 系统未声明执行成功
- **并且** 要么返回诚实的“不支持”响应，要么创建一个澄清任务
- **并且** 未启动具有无限制工作区访问权限的适配器运行。

### Requirement: 会话上下文包
系统 MUST 构建一个会话作用域的上下文包，用于代理执行，其中包含当前目标、最近消息、执行账本、选定的制品、最新的差异元数据、已更改的文件、preview/deploy 状态，以及相关的合约数据（如果存在）。

#### Scenario: 为后续任务构建 Agent 指令
- **WHEN** 在同一会话中已有先前制品存在后，启动一个可执行任务
- **THEN** 适配器指令包含该会话的结构化上下文包
- **且** 当用户选中某个制品时，该包标识出所选制品的上下文
- **且** 该包包含最新差异、变更文件、预览健康状态、模拟部署状态以及当前目标（若可用）
- **且** 该包排除来自其他会话的记录。

### Requirement: 基于角色的指令构建器
系统 MUST 为前端、后端、QA/review 和管理员任务构建角色特定的适配器指令，同时保留安全边界和有意义的用户意图。

#### Scenario: 前端指令已生成
- **WHEN** 前端可执行任务开始
- **THEN** 指令针对演示前端工作区
- **并且** 包含原始用户请求及相关会话上下文
- **并且** 保留受保护路径和依赖安装的防护规则
- **并且** 未被重写为无关的仅登录页面指令

#### Scenario: 后端指令已生成
- **WHEN** 后端可执行任务启动
- **THEN** 指令指向安全演示后端工作区
- **且** 包含相关合约与前端集成上下文
- **且** 不允许自由修改 AgentHub 平台后端。

#### Scenario: 生成 QA 或审查指令
- **WHEN** 一个 QA 或审查可执行任务开始
- **THEN** 指令聚焦于差异、契约、风险、变更文件以及证据
- **并且** 除非后续明确策略引入阻断门控，否则生成的审查结果仍为建议性质。

### Requirement: 演示后端目标
系统 MUST 为后端代理工作提供或规划一个安全的演示后端目标，以便在不修改 AgentHub 控制平面后端的情况下评估后端执行。

#### Scenario: 后端 Agent 任务已创建
- **WHEN** 系统创建一个后端执行任务
- **THEN** 该任务针对演示后端目标，例如 `apps/demo-api`
- **并且** 受保护的平台路径仍保持防护
- **并且** 生成的差异通过现有制品流程收集。

### Requirement: 合约优先编排器
系统 MUST 支持针对有界全栈迷你应用请求的契约优先编排，在前端和后端实现任务之前生成共享的应用契约。

#### Scenario: 编排器收到有界迷你应用请求
- **WHEN** 用户请求一个受支持的迷你应用，例如待办事项、笔记或迷你 CRM 联系人，无论是否 `@orchestrator` 或未明确提及角色
- **THEN** 系统创建一个结构化的应用契约
- **并且** 该契约定义了目标、实体、API 端点、UI 界面、目标目录和验收标准
- **并且** 后端和前端任务引用同一份契约
- **并且** 同一会话中的写入任务保持串行执行，除非存在冲突处理机制

### Requirement: 有界全栈垂直切片
系统 MUST 验证一条从用户需求出发，经过合约、后端任务、前端任务、QA/review、差异对比、预览和模拟部署的完整全栈小型应用路径。

#### Scenario: 迷你应用垂直切片进行演练
- **WHEN** P6 E2E 演练运行一个受支持的迷你应用请求
- **THEN** 证据记录了需求、契约、任务 ID、任务运行 ID、适配器类型、变更文件、差异制品、审查制品、预览状态和模拟部署状态
- **且** 仅当实际运行时才记录真实的 Claude/Codex 执行
- **且** 兜底或脚本化行为需如实标注
- **且** 模拟部署仍标注为模拟。

### Requirement: 执行安全性与诚实性
系统 MUST 对于不支持的请求应诚实失败或请求澄清，
并且 MUST 为所有直接执行和编排执行的任务
保留安全的目标边界。

#### Scenario: 用户请求生成不支持的任意应用
- **WHEN** 用户请求了不支持的任意 SaaS、模糊的分配、生产环境部署、payment/auth/multi-tenant 系统或不受限的平台代码变更
- **THEN** 系统不会创建不受限的编码任务
- **并且** 系统会解释不支持的边界或要求提供有界目标
- **并且** 系统会保留 P4/P5 基线行为。
