## 1. P6 Agent 执行能力升级

- [x] 1.1 P6-1 消息路由、直接角色分配与编排器自动运行 v1。
  - 目标：默认将未提及角色的消息路由至编排器/管理器，同时提供 `@frontend`、`@backend`、`@qa` 和 `@review` 的显式分配快捷方式，并自动启动由编排器创建的安全演示目标任务。
  - 范围：
    - 实现路由优先级：优先处理显式提及，否则路由至编排器/管理器；
    - 将 `@orchestrator` 和未提及角色的消息路由至编排器/管理器；
    - 扩展显式角色提及功能，为支持的限定请求创建直接角色任务；
    - 通过现有 TaskRun 路径自动启动由编排器创建的安全演示目标编码任务；
    - 当未显式提及角色时，默认将选定的制品上下文传递给编排器；
    - 在任务中保留用户的原始请求；
    - 将任务分配给正确的现有角色或审核角色表示；
    - 保持任务可通过现有 TaskRun 路径启动；
    - 诚实地拒绝或澄清不支持的宽泛请求；
    - 保留现有 `@orchestrator` 登录页面和 P5 规划器行为。
  - 验收标准：
    - 未显式提及角色的消息路由至编排器/管理器；
    - 编排器能够回答问题、创建任务、生成 contract/task 图表、请求澄清或诚实地拒绝不支持的请求；
    - 由编排器创建的安全演示目标前端任务自动启动；
    - 直接的 `@frontend` 请求可创建可执行的前端任务；
    - 直接的 `@backend` 请求可创建可执行的后端任务，且仅限安全演示后端范围（一旦可用或明确为此阶段准备）；
    - 直接的 `@qa` 和 `@review` 请求创建只读的 review/QA 任务；
    - 原始用户请求被存储且可见；
    - 没有适配器获得不受限制的工作区访问权限。
  - 验证：
    - 后端提及路由测试；
    - 前端任务渲染测试（若 UI 副本有变更）；
    - `pnpm check`；
    - `pnpm test`；
    - `git diff --check`；
    - `openspec validate agenthub-p6-agent-execution-upgrade --strict`。

- [x] 1.2 P6-2 会话上下文包。
  - 目标：为智能体运行提供结构化的会话上下文，以支持多轮迭代。
  - 范围：
    - 仅基于当前会话记录构建上下文包服务；
    - 包含最近的消息、执行账本、选定的制品、最新的差异元数据、已变更文件、preview/deploy 状态、当前目标以及应用契约（若存在）；
    - 将选定的制品上下文传递给编排器，用于默认路由的消息；
    - 在适配器运行请求中包含上下文包元数据；
    - 验证选定的制品引用是否与当前会话匹配；
    - 防止跨会话上下文泄漏。
  - 验收标准：
    - 后续任务能接收到相关的先前上下文；
    - 选定的制品上下文在可用时能被呈现；
    - 上下文包排除其他会话；
    - 现有的 P4/P5 任务执行仍能正常工作。
  - 验证：
    - 后端上下文包单元测试；
    - 任务运行请求测试；
    - `pnpm check`；
    - `pnpm test`；
    - `git diff --check`；
    - `openspec validate agenthub-p6-agent-execution-upgrade --strict`。

- [x] 1.3 P6-3 基于角色的 Agent 指令构建器。

  - 范围：
    - 引入指令构建器边界；

  - 验收标准：

## 2. P6 的明确非目标

- 任意 SaaS 生成。
- 无限制编辑 AgentHub 平台代码。
- 生产环境部署。
- 多用户即时通讯。
- Matrix、飞书、微信、Slack 或其他外部 IM 集成。
- 提供商市场。
- Docker 沙箱。
- PR 创建。
- 企业审批工作流。
- 支付、认证或多租户生产系统。

## 3. P6 完成定义

- 直接提及会为受支持的角色请求创建可执行任务。
- 未明确提及角色的消息默认路由至编排器/管理员。
- 明确提及角色具有最高优先级，并作为高级分配快捷方式。
- 编排器创建的安全演示目标任务可通过现有 TaskRun 路径自动启动。
- 直接任务保留原始用户请求。
- 代理指令包含会话上下文和（若可用）选定的制品上下文。
- 前端、后端、QA/review 和管理员指令按角色区分。
- 后端执行目标为安全演示后端，而非 AgentHub 平台 API。
- 编排器可为单一有界全栈迷你应用族生成共享契约。
- 迷你应用垂直切片包含差异、审查、预览和模拟部署证据。
- 仅在实际运行时记录真实的 Claude/Codex 成功状态。
- 不受支持的请求会如实失败或创建澄清任务。
- P4/P5 基线和现有适配器保持不变。
