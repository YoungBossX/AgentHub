## ADDED Requirements
### Requirement: 部署提供者抽象
系统 MUST 定义了用于预发布和模拟部署路径的部署提供程序抽象。

#### Scenario: 部署提供者结果记录标准元数据
- **WHEN** 部署提供者创建部署结果
- **THEN** 结果 MUST 包含提供者 ID、提供者类型、目标 ID、构建命令、部署命令、可用时的输出 URL、状态和日志。

#### Scenario: 现有模拟部署保持可用
- **WHEN** 任务继续使用模拟部署路径
- **THEN** 系统 MUST 保留模拟部署行为
- **并且** 部署制品 MUST 保持明确标记为模拟。

#### Scenario: 请求了未知的提供商
- **WHEN** 部署请求引用了未知的提供者 ID
- **THEN** 系统 MUST 诚实地拒绝该请求
- **并且** 它 MUST 不会创建成功的部署制品。

### Requirement: 目标感知的部署配置
系统 MUST 从目标注册表中读取了暂存部署配置。

#### Scenario: 内置演示前端具有预发布部署配置
- **WHEN** 请求为内置演示前端目标执行暂存部署
- **THEN** 系统 MUST 从目标注册表中解析构建命令、输出目录、服务命令或服务策略、允许路径、禁止路径以及提供商可用性。

#### Scenario: 外部前端目标具有部署配置
- **WHEN** 请求对已注册的外部前端目标执行暂存部署
- **THEN** 系统 MUST 从该外部目标的注册信息或分析器结果中解析部署元数据
- **并且** 系统 MUST 应用该外部目标的允许与拒绝路径策略。

#### Scenario: 目标缺少部署配置
- **WHEN** 目标没有足够的部署配置用于预发布环境部署
- **THEN** 系统 MUST 返回了不受支持或失败的部署结果
- **并且** 系统 MUST 不得猜测不安全的构建或服务命令。

### Requirement: 本地暂存部署提供者
系统 MUST 提供了一个本地暂存部署提供程序，用于在本地构建并提供前端输出。

#### Scenario: 本地预发布部署成功
- **WHEN** 部署门禁通过，达到可部署的前端目标
- **THEN** 本地预发布提供者 MUST 运行配置的构建命令
- **并且** MUST 在本地提供配置的输出目录服务
- **并且** MUST 生成真实的本地预发布 URL
- **并且** MUST 验证预发布 URL 可访问
- **并且** MUST 记录就绪的预发布部署制品。

#### Scenario: 构建命令失败
- **WHEN** 配置的构建命令以失败退出
- **THEN** 暂存部署 MUST 被标记为失败
- **并且** 构建日志 MUST 被记录
- **并且** 系统 MUST 不声明就绪的暂存 URL。

#### Scenario: 输出目录缺失
- **WHEN** 构建命令执行完成，但配置的输出目录缺失
- **THEN** 暂存部署 MUST 被标记为失败
- **并且** 失败原因 MUST 指明缺失的输出目录名称。

#### Scenario: 预发布服务器被取消
- **WHEN** 用户或清理操作取消正在运行的预发布部署
- **THEN** 部署状态 MUST 变为已取消
- **并且** 本地服务进程 MUST 在追踪时被停止。

### Requirement: 部署日志与状态制品
系统 MUST 持久化存储预发布部署状态、日志及源制品元数据。

#### Scenario: 部署状态转换被记录
- **WHEN** 一个预发布部署正在推进
- **THEN** 状态转换 MUST 使用 `queued`、`building`、`deploying`、
  `ready`、`failed` 或 `cancelled`
- **并且** 每次转换 MUST 记录在部署制品元数据或
  TaskRunEvent 日志中。

#### Scenario: 部署日志可见
- **WHEN** 预发布部署会运行构建或服务步骤
- **THEN** 系统 MUST 记录标准输出、标准错误、命令退出码以及高级状态消息，且不暴露机密信息。

#### Scenario: 部署制品链接源证据
- **WHEN** 创建了一个预发布部署制品
- **THEN** 它 MUST 引用目标 ID、源 TaskRun、源差异制品（如有）、审查制品（如有）以及预览（如有）。

### Requirement: 部署 Gate
系统 MUST 门控在审查、预览和目标策略上执行分阶段部署，以确保安全性。

#### Scenario: 审核失败
- **WHEN** 源差异的最新相关审核已失败
- **THEN** 预发布环境部署 MUST 被阻止
- **并且** 部署结果 MUST 记录该审核失败阻止了部署。

#### Scenario: 预览失败
- **WHEN** 存在预览前置条件且预览健康状态为失败或不健康
- **THEN** 预发布部署 MUST 被阻止
- **并且** 部署结果 MUST 记录预览健康状态阻止了部署。

#### Scenario: 存在目标策略违规
- **WHEN** 审查、调度器元数据或目标策略证据显示存在
  拒绝路径或平台代码违规
- **THEN** 预发布部署 MUST 被阻止
- **并且** 系统 MUST 不得将输出标记为就绪状态。

#### Scenario: 请求生产部署
- **WHEN** 部署请求要求使用生产环境或类生产环境
- **THEN** P11 MUST 拒绝该请求
- **并且** 它 MUST 不执行构建或部署命令。

### Requirement: P11 预演与基线保留
系统 MUST 演练真实的分阶段部署行为，并保留现有的 AgentHub 基线。

#### Scenario: 内置或外部前端预发布演练成功
- **WHEN** P11 针对内置演示前端或外部 Vite 示例目标进行演练
- **THEN** 预发布部署 MUST 生成可访问的 URL
- **并且** 日志、状态、目标元数据和部署制品 MUST 被记录。

#### Scenario: 现有基线保持不变
- **WHEN** P11 冻结评审已完成
- **THEN** P6 有界全栈执行、P7 目标注册表策略、P8 scheduler/locks、P9 外部工作区模式以及 P10 鲁棒性行为 MUST 仍保持可运行状态。

#### Scenario: 请求了超出范围的部署能力
- **WHEN** 请求需要生产环境部署、云服务商令牌管理、Docker/Kubernetes、域名管理、数据库托管、自动回滚或多用户部署审批
- **THEN** P11 MUST 如实拒绝或推迟该请求。
