## 为什么

P10 使 AgentHub 在本地和外部项目执行时更加安全，但部署仍由模拟后端支持。AgentHub 可以生成、审查、预览、验证和记录变更，但无法生成用户可作为已部署构建打开的真实预发布部署制品。

P11 添加了一条真实、面向目标环境的预发布部署路径，同时明确将生产部署排除在范围之外。

## 变更内容

- 为预发布部署提供者添加部署提供者抽象层，包含提供者 ID、提供者类型、目标 ID、构建命令、部署命令、输出 URL、状态和日志。
- 从目标注册表中读取内置演示前端和已注册外部前端目标的预发布 deploy/build 配置。
- 添加本地预发布部署提供者，该提供者运行允许的构建命令，在本地提供构建输出目录，生成可访问的预发布 URL，并记录真实的预发布部署制品。
- 添加部署日志和状态制品，包含 `queued`、`building`、`deploying`、`ready`、`failed` 和 `cancelled` 状态。
- 添加部署门控，当审查失败、预览失败或存在目标策略违规时，预发布部署不会运行。
- 保留现有模拟部署行为，在仍需要用于旧版演示和兜底路径的地方继续使用。
- 针对内置演示前端或外部 Vite 示例目标进行 P11 演练，并验证 URL 可达性、日志、制品和 P6/P7/P8/P9/P10 基线。

## 能力

### 新能力

- `staging-deploy`: 目标感知的真实预发布部署提供程序、部署配置、部署门控、status/log 制品、本地预发布 URL 服务以及冻结演练。

### 修改后的能力

P11 引入了暂存部署能力，同时保留了现有的模拟部署行为和现有的 preview/deploy 基线合约。

## 影响

P11 后续应用时的预期实现影响：

- 后端：
  - 部署提供者接口与提供者选择；
  - 从目标注册表获取的、感知目标的 deploy/build 配置；
  - 本地暂存构建与静态服务；
  - 部署 status/log 制品持久化；
  - 针对审查、预览、目标策略及生产部署禁止的部署门禁；
  - 按需更新暂存部署 creation/list/read 的 API。
- 前端：
  - 暂存部署制品 cards/status 展示；
  - 部署 log/status 可见性；
  - 与模拟和生产环境明确区分的暂存标签。
- 目标注册表：
  - 针对内置演示前端及已配置的外部前端目标的暂存 build/deploy/output 元数据。
- 运行时：
  - 仅允许显式白名单中的本地 build/serve 命令；
  - 不支持云部署、生产部署、令牌管理、Docker/Kubernetes、域名管理、数据库托管、自动回滚或多用户部署审批。
