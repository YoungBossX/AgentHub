## 为什么

外部目标注册目前依赖用户手动输入绝对本地路径并手动选择有界允许路径。这会拖慢演示使用速度，且让注册任意本地文件夹的方式变得不清晰。

## 变更内容

- 为运行时工作区设置添加轻量级本地文件夹选择器。
- 添加后端 API，用于列出子目录以实现安全的本地文件夹导航。
- 将默认文件夹浏览设置为实用的本地起始位置，例如桌面、文档和 AgentHub 工作区附近。
- 允许为任何现有文件夹注册外部目标，无需识别项目类型。
- 将所选文件夹视为可写目标范围，同时保留被禁止的受保护路径，例如 `.git`、`.env` 和 `node_modules`。
- 移除用户在运行时设置界面中手动输入允许写入路径的需求。

### 修改后的能力

- `external-workspace`: 注册用户体验与策略变更，允许选择任意本地文件夹，并通过受保护路径拒绝机制授予所选文件夹作用域。

## 影响

- 后端：
  - 本地文件夹列表 API；
  - 外部目标注册的允许路径策略，限定于所选文件夹范围。
- 前端：
  - 运行时设置文件夹选择器对话框；
  - 注册请求默认使用所选文件夹范围。
- 测试：
  - 后端 API 与注册测试；
  - 前端运行时设置测试。
- 文档：
  - `docs/change-log.md` 更新。
