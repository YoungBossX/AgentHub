## 背景

P9 引入了具有显式根目录和允许路径的外部项目目标。
当前的运行时设置界面要求用户输入路径和允许路径。
对于本地演示使用，用户需要更快捷的方式找到文件夹，且此更改不应要求该文件夹被识别为 Vite、React、后端或其他项目类型。

## 目标 / 非目标

**目标：**

- 允许用户在运行时设置中浏览本地文件夹并选择目录。
- 将任何现有文件夹注册为外部目标，无需进行项目分析。
- 使所选文件夹成为目标范围，同时保持现有的受保护拒绝路径强制执行。
- 保持实现本地化、最小化，并与当前的 Next.js + FastAPI 技术栈兼容。

**非目标：**

- 添加 Electron、Tauri 或系统原生文件打开对话框。
- 安装依赖项。
- 检测项目类型或推断命令。
- 授予对不安全宽泛根目录（如文件系统根目录、home、Desktop、Documents、Downloads 或系统目录）的访问权限。
- 削弱对 `.git`、`.env`、`node_modules` 或生成的 dependency/build 输出的拒绝路径保护。

## 决策

- 使用后端驱动的文件夹选择器替代浏览器原生目录上传。浏览器原生文件夹选择无法可靠地暴露绝对本地路径，而 AgentHub 已拥有一个可列出目录的本地 FastAPI 后端。
- 仅暴露导航所需的目录元数据：当前路径、父路径、命名起始位置以及可见子文件夹。该 API 不会读取文件或分析项目内容。
- 使用显式允许路径 `*` 表示所选文件夹的写入范围。现有注册已拒绝 `.`，以防止意外授予过宽权限。`*` 仍为有意为之，可跨序列化保留，并能被目标路径策略解释为“在已注册根目录内，排除被拒绝路径”。
- 保留不安全的根目录验证。用户可以浏览桌面或文档，但必须在注册成功前选择更具体的文件夹。
- 保持注册表单紧凑：路径输入加上“选择文件夹”和“注册”。旧的允许路径输入已被移除，因为所选文件夹本身即为范围。

## 风险 / 权衡

- 宽泛的选定文件夹范围可能允许比之前仅 `src` 注册更多的文件 -> 通过要求显式指定文件夹根目录、继续拒绝宽泛根目录以及保留被拒绝的路径来缓解。
- 列出本地目录可能会在 UI 中暴露文件夹名称 -> 通过在本地单用户应用中仅返回本地用户请求的目录 names/paths 来缓解。
- 某些文件夹可能无法读取 -> 通过返回 400 级别错误并显示状态消息（而非导致整个设置页面失败）来缓解。
