## ADDED Requirements
### Requirement: 用于外部目标注册的本地文件夹选择器
系统 MUST 提供了一个轻量级的本地文件夹选择器，用于在运行时设置中注册外部目标。

#### Scenario: 列出文件夹起始位置
- **WHEN** 运行时设置页面请求本地文件夹起始位置
- **THEN** 系统 MUST 返回可访问的起始文件夹，包括桌面、文档以及 AgentHub 工作区附近（若存在）。

#### Scenario: 列出文件夹子项
- **WHEN** 用户导航至一个已有的可读文件夹
- **THEN** 系统 MUST 返回子文件夹供导航
- **并且** 它 MUST 不要求所选文件夹包含项目标记文件，例如 `package.json`、`src` 或后端应用文件。

#### Scenario: 选择文件夹后填充项目路径
- **WHEN** 用户在运行时设置中选择一个文件夹
- **THEN** 所选文件夹的绝对路径 MUST 填充外部目标项目路径字段。

### Requirement: 选定文件夹范围注册
系统 MUST 允许外部目标注册，以使用所选文件夹作为写入范围，同时保留受保护路径的拒绝规则。

#### Scenario: 选中的文件夹在未手动添加允许路径的情况下注册
- **WHEN** 用户通过运行时设置注册一个现有的非广播本地文件夹
- **THEN** 注册请求 MUST 不要求手动输入允许路径
- **并且** 存储的外部目标 MUST 表示所选文件夹范围。

#### Scenario: 受保护路径仍被拒绝
- **WHEN** 外部目标以选定文件夹作用域注册
- **THEN** 拒绝路径 MUST 包含受保护路径，例如 `.env`、`.env.*`、
  `secrets`、`.git`、`node_modules`、依赖目录以及生成的构建
  输出目录。

#### Scenario: 不安全的宽泛根路径仍被拒绝
- **WHEN** 用户尝试直接注册文件系统根目录、主目录、桌面、文档、下载或系统目录
- **THEN** 系统 MUST 拒绝该注册
- **并且** 拒绝时 MUST 说明必须选择更具体的文件夹。
