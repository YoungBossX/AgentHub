## ADDED Requirements
### Requirement: 日常工作区会话导航
系统 SHALL 提供一个日常使用的会话导航界面，让用户无需离开主工作区即可浏览、创建、选择、搜索和了解最近的 AgentHub 会话。

#### Scenario: 搜索筛选可见会话
- **WHEN** 用户在会话搜索控件中输入文本
- **THEN** 会话列表仅显示标题或可见摘要与搜索文本匹配的会话

#### Scenario: 空搜索仍可理解
- **WHEN** 用户搜索与任何会话都不匹配的文本
- **THEN** 会话列表显示空状态消息，且不清除已选中的会话

#### Scenario: 新会话保持可用
- **WHEN** 用户从工作区侧边栏创建新会话
- **THEN** 新会话被添加到列表中并成为当前选中的会话

### Requirement: 会话模式可见性
系统 SHALL 使直接和群组会话框架在工作区中可见，
同时保留现有的提及路由和编排器行为。

#### Scenario: Direct 模式可见
- **WHEN** 工作区处于 Direct 模式
- **THEN** 聊天头部或侧边栏标识了当前聚焦的对话模式
  且用户仍可发送普通消息

#### Scenario: 群组模式可见
- **WHEN** 工作区处于群组模式
- **THEN** 工作区将编排器和活跃角色智能体显示为协调对话，而不改变调度器或计划验证器的行为

#### Scenario: 模式显示不创建任务
- **WHEN** 用户切换或查看对话模式，但未发送编码请求
- **THEN** 系统不会创建 Task 或 TaskRun

### Requirement: 消息上下文管理
系统 SHALL 允许用户在发送后续请求之前，附加、查看和移除消息或制品上下文。

#### Scenario: 将消息引用到编辑器
- **WHEN** 用户引用一条消息
- **THEN** 编辑器在发送前将引用的消息内容显示为待处理上下文

#### Scenario: 将制品用作上下文
- **WHEN** 用户选择一个制品作为上下文
- **THEN** 编辑器在发送前显示所选制品的类型和标题

#### Scenario: 发送前清除上下文
- **WHEN** 用户清除选中的上下文
- **THEN** 下一条发送的消息不包含该上下文引用

### Requirement: 只读计划审查
系统 SHALL 在工作区中、任务执行之前或同时，公开规划器推理和任务审查元数据，但不允许在 P20 阶段修改计划。

#### Scenario: 计划元数据可见
- **WHEN** 任务包含计划审核元数据
- **THEN** 任务时间线显示依据、分配角色、目标、
  依赖项、计划文件、验收标准和验证预期
  （若可用）

#### Scenario: 只读边界被保留
- **WHEN** 计划审查元数据已显示
- **THEN** 工作区不提供在未来的已批准计划编辑阶段之前修改计划的控件

#### Scenario: 规划器兜底证据可见
- **WHEN** 任务通过审计的规划器兜底创建
- **THEN** 工作区在用户可读的任务或任务上下文中显示兜底来源或原因

### Requirement: Agent 与证据状态摘要
系统 SHALL 在不要求用户检查原始任务追踪 JSON 的情况下，汇总工作区中的活跃 Agent、provider/runtime 状态、内存快照、选定目标及最新证据。

#### Scenario: 活跃代理被汇总
- **WHEN** 会话中有任务或任务运行被分配给角色代理
- **THEN** 工作区摘要显示活跃的代理角色以及提供者或适配器元数据（如果可用）

#### Scenario: 目标和内存上下文已汇总
- **WHEN** 会话已选定目标或内存快照
- **THEN** 工作区在紧凑的上下文区域中显示活动目标和内存快照状态

#### Scenario: 最新证据已汇总
- **WHEN** 存在差异、审查、预览或部署制品
- **THEN** 工作区显示最新证据状态，并提供通往制品面板的路径

### Requirement: 制品导航工作台
系统 SHALL 使 Diff、Review、Preview 和 Deployment 制品能够从任务时间线和制品面板中轻松发现和检查。

#### Scenario: 从任务时间线中选择制品
- **WHEN** 任务运行已生成制品
- **THEN** 用户可选择制品，制品面板将打开该制品的详情

#### Scenario: 制品面板保持类型导航
- **WHEN** 存在多种制品类型
- **THEN** 制品面板允许用户在 Diff、Preview、Review 和 Deployment 制品之间切换，而不会丢失已选中的会话

#### Scenario: 制品上下文交接保持可用
- **WHEN** 用户选择将制品用作上下文
- **THEN** 编排器接收到制品上下文，下一条消息可引用该上下文

### Requirement: 工作区设置分离
系统 SHALL 将详细的提供商、目标、记忆和联系人配置保留在主聊天界面之外，同时确保这些设置可从主工作区访问。

#### Scenario: 设置入口保持可访问
- **WHEN** 用户需要配置联系人、运行时、工作区目标或内存
- **THEN** 侧边栏提供指向相应设置页面的清晰链接

#### Scenario: 聊天界面保持聚焦
- **WHEN** 用户正在聊天或审查任务证据
- **THEN** 详细的 runtime/provider/target 表单未嵌入主聊天线程中
