## 背景

P23 新增了制品工作台，生成的制品可以预览、作为版本进行编辑，并在后续消息中引用。AgentHub 已有本地预览和预发布部署证据，但部署目前仍主要停留在 mock/local 卡片层面，尚未形成清晰的、感知提供方的工作流。

日常使用需要部署状态卡片，能够展示提供商选择、配置状态、部署尝试、日志、目标 URL 及后续操作，而无需在缺少提供商凭据或审批时，假装已发生生产环境的第三方部署。

## 目标

- 为本地暂存和外部第三方交接提供商添加受控部署提供商注册表。
- 允许 AgentHub 根据预览或制品证据创建可审计的部署 request/status 卡片。
- 展示提供商配置状态，但不暴露密钥。
- 保留本地暂存部署行为及现有基于模拟的部署卡片。
- 允许在后续聊天上下文中引用部署卡片。
- 除非提供商已配置并明确批准，否则保持生产环境第三方部署执行处于禁用状态。

## 非目标

- 无云密钥管理器。
- 默认不进行生产部署。
- 无提供商市场。
- 无任意 shell 命令部署提供商。
- 不替代 PreviewService 或现有 Deployment 制品。
- 无多用户 RBAC 或企业审计后端。
- 无 PR 创建或远程仓库发布。

## 安全

部署提供者必须明确且可审计。缺失凭据、不支持的提供者目标、构建失败或未经批准的生产部署必须生成真实的 failed/blocked 状态卡片，而非声称成功。

目标注册表、计划验证器、调度器和护栏仍作为执行边界。部署卡片可引导后续操作，但不得绕过任务规划或目标范围内的权限。
