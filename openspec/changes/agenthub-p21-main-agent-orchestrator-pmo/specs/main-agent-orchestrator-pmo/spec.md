## ADDED Requirements
### Requirement: PMO 计划决策状态
系统 SHALL 记录可审计的 PMO 计划决策状态，用于需要用户在执行前审查的任务或任务组。

#### Scenario: 计划需要审核
- **WHEN** 已验证的计划被标记为需要用户审核
- **THEN** 系统记录待定的 PMO 决策状态，而不启动 TaskRun

#### Scenario: 计划决策可见
- **WHEN** 任务包含 PMO 决策元数据
- **THEN** 任务响应或任务轨迹暴露决策状态、原因、
  审查人、时间戳以及下一步行动摘要

### Requirement: 计划决策操作
系统 SHALL 提供有限的操作，用于批准、拒绝或请求澄清 PMO 已评审的计划，而不接受任意未经验证的计划变更。

#### Scenario: 审批已审核的计划
- **WHEN** 用户审批一个待处理的已审核计划
- **THEN** 该任务将符合常规调度器就绪评估条件
  且执行仍沿用现有的 TaskRun 创建路径

#### Scenario: 拒绝已审核的计划
- **WHEN** 用户拒绝一个待审核的计划
- **THEN** 该任务被标记为阻塞或拒绝，并附上用户原因，且不会创建 TaskRun

#### Scenario: 请求澄清
- **WHEN** 用户请求对待审核的计划进行澄清
- **THEN** 系统记录“需要澄清”状态，并展示面向用户的下一步操作，而非执行任务

#### Scenario: 原始计划变更被拒绝
- **WHEN** 客户端尝试通过 PMO 决策操作对任意计划 JSON 进行变更
- **THEN** 系统拒绝该变更或忽略不支持的字段

### Requirement: 任务图协调可见性
系统 SHALL 应暴露可供 PMO 读取的任务图就绪组，这些组源自现有的调度器和依赖状态。

#### Scenario: 可并行执行的任务可见
- **WHEN** 多个任务已满足依赖关系且目标锁可用
- **THEN** 任务追踪或任务界面将其标识为可并行运行

#### Scenario: 等待依赖可见
- **WHEN** 任务正在等待依赖完成
- **THEN** 系统显示阻塞的依赖 ID 及原因

#### Scenario: 等待目标锁可见
- **WHEN** 一个任务正在等待目标写锁
- **THEN** 系统显示目标 ID、锁持有者运行 ID 及原因

### Requirement: PMO 恢复指引
系统 SHALL 针对失败、中断、可重试、可兜底、需审批以及过期的 TaskRun，暴露用户可读的下一步操作。

#### Scenario: 可重试失败
- **WHEN** TaskRun 在可重试状态下失败
- **THEN** 系统暴露一个重试下一步操作，该操作与现有的重试端点绑定

#### Scenario: 显式兜底
- **WHEN** 兜底重试可用
- **THEN** 系统将兜底作为显式审计操作暴露，且不会静默替换提供商

#### Scenario: 需要审批
- **WHEN** TaskRun 等待审批
- **THEN** 系统通过 risk/reason 上下文暴露批准和拒绝的下一步操作

### Requirement: 冲突与阻塞说明
系统 SHALL 需以 PMO 可读的术语解释代码冲突、脏工作区阻塞、目标锁定阻塞及验证阻塞，且不自动合并或丢弃变更。

#### Scenario: 脏工作树阻塞器
- **WHEN** 调度器检测到安全 planned/dependency 文件之外的脏文件
- **THEN** PMO 摘要列出受影响文件，并提供检查或手动解决指导

#### Scenario: 冲突未自动合并
- **WHEN** 两个任务在重叠文件或目标锁上发生冲突
- **THEN** 系统不会自动合并、自动暂存或丢弃更改

#### Scenario: 验证阻塞
- **WHEN** PlanValidator 或目标策略拒绝某个计划
- **THEN** PMO 摘要显示安全错误摘要，且无可执行任务绕过验证

### Requirement: PMO 任务追踪证据
系统 SHALL 在任务追踪或任务元数据中记录 PMO 协调证据，以便用户审计任务为何被批准、阻塞、重试或兜底。

#### Scenario: 决策证据出现在任务轨迹中
- **WHEN** 记录了一次 PMO 决策
- **THEN** 任务轨迹包含决策状态、原因、执行者、时间戳以及
  相关任务 ID

#### Scenario: 恢复证据出现在任务追踪中
- **WHEN** 显示重试或兜底指导
- **THEN** 任务追踪包含源状态和推荐操作
  但不声称执行成功

#### Scenario: 密钥保持隐藏
- **WHEN** PMO 证据包含 provider/runtime 上下文
- **THEN** 原始 API 密钥、受保护的主机路径和密钥不会被暴露
