## 为什么

AgentHub 现在已具备会话、PMO 风格编排器、Agent 目录、制品工作台和部署状态卡片。日常使用中尚存的差距在于控制：用户需要一种简单的方式，在向编排器或角色 Agent 请求后续操作之前，将消息片段、制品章节、部署证据以及先前的输出收集到一个显式的上下文托盘中。

没有这一点，第二轮工作会显得脆弱：上下文是隐式的，选中的项目容易丢失，系统也无法清晰显示哪些证据传递给了规划器，哪些仅停留在 UI 中可见。

## 变更内容

- 为选中的消息、制品、制品版本、选中的 text/sections、部署证据和简短用户备注添加上下文托盘模型。
- 为规划器可见的上下文添加安全的上下文项标准化与脱敏处理。
- 允许用户在聊天工作区中添加、移除、重新排序、清空和发送上下文项，而无需将聊天页面变成配置界面。
- 通过将当前单个选中制品和引用消息的行为映射到新的托盘模型中，保留现有行为。
- 在消息上下文、规划器证据、任务追踪或 API 响应中（视情况而定）暴露上下文交接证据。
- 为上下文托盘标准化、脱敏、载荷结构、规划器交接及 UI 操作添加针对性测试。

## 能力

### 新能力

- `context-control-secondary-interaction`: 涵盖显式上下文托盘选择、安全上下文标准化、后续负载、规划器交接以及第二轮工作的 UI 操作。

### 修改后的能力

- 无。

## 影响

- `apps/api`：消息上下文 schema/helpers、上下文 normalization/redaction、
  会话上下文包/规划器可见数据中的上下文证据、测试。
- `apps/web`：上下文托盘 UI、message/artifact/deployment 上下文操作、
  编排器负载行为、测试。
- 现有的 P20-P24 聊天、制品、部署、规划器、调度器、目标
  注册表和护栏行为保持兼容。
