## ADDED Requirements
### Requirement: 上下文托盘负载
AgentHub SHALL 支持为后续消息提供显式的上下文托盘负载。

#### Scenario: 发送多个上下文项
- **WHEN** 用户在上下文托盘中发送包含选中消息、制品、选中文本、部署证据或用户笔记的消息
- **THEN** 消息上下文包含一个标准化的 `contextItems` 列表
- **并且** 现有的 `selectedArtifactId` 和 `quotedMessage` 字段保持兼容

#### Scenario: 上下文条目限制
- **WHEN** 上下文托盘负载超出配置的条目或文本限制
- **THEN** AgentHub 诚实地截断或拒绝不安全的多余内容
- **并且** 不会静默地将无界内容传递给提供商

### Requirement: 安全上下文标准化
AgentHub SHALL 在上下文项进入规划器可见上下文之前，对其进行标准化和脱敏处理。

#### Scenario: 编辑不安全的值
- **WHEN** 上下文包含受保护的路径、类似密钥的键、原始 API 令牌，
  或包含类似密钥文本的提供者日志
- **THEN** 规划器可见的上下文省略或编辑了这些值
- **并且** 编辑后的上下文仍可审计

#### Scenario: 部署上下文
- **WHEN** 部署卡片作为上下文使用
- **THEN** 规划器可见的上下文包括：提供者 ID、提供者类型、状态、
  环境、可用时的 URL、源制品 ID 以及安全日志摘要
- **并且** 除非有证据表明该提供者确实成功，否则不得声称第三方生产环境成功

### Requirement: 上下文托盘 UI
AgentHub SHALL 让用户能够从聊天工作区管理后续上下文。

#### Scenario: 发送前管理上下文
- **WHEN** 用户添加消息、制品、部署、选中文本或笔记上下文
- **THEN** 编辑器显示一个紧凑的上下文托盘，包含移除、重新排序和清除控件

#### Scenario: 保持聊天简洁性
- **WHEN** 上下文托盘为空
- **THEN** 聊天工作区保持聚焦于会话消息，不显示运行时配置控件

### Requirement: 基于上下文的后续路由
AgentHub SHALL 通过现有的对话和规划管道路由具有上下文感知的后续消息。

#### Scenario: 上下文感知的编码跟进
- **WHEN** 一条带有上下文的跟进消息要求修改代码或制品
- **THEN** 对话路由器/规划器接收安全的上下文摘要
- **并且** 任何可执行的任务仍需通过 PlanValidator、Scheduler 和编码代理

#### Scenario: 纯上下文聊天
- **WHEN** 带上下文的后续消息仅用于对话
- **THEN** AgentHub 可能会创建编排器助手回复
- **且** 纯聊天不会创建 TaskRun

### Requirement: 上下文证据
AgentHub SHALL 使上下文交接对审计和调试可见。

#### Scenario: 任务追踪记录上下文
- **WHEN** 带有上下文的消息会生成规划器证据或任务上下文
- **THEN** AgentHub 记录上下文条目数量、条目类型、所选制品 ID、所选版本 ID、部署 ID，并在适当时记录脱敏状态
- **且** 原始密钥或受保护的主机路径不会被记录
