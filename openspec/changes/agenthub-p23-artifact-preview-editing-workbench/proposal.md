## 为什么

AgentHub 已能生成差异、审查、预览和部署证据，但制品仍像静态卡片而非可编辑的工作产物。用户需要一个专注的制品工作台，用于检查、轻量编辑、版本管理、引用生成的网页、文档、代码片段、差异和部署证据，并将其交还给智能体。

## 变更内容

- 新增制品工作台基础功能，支持内联预览、聚焦检查、轻量编辑、版本历史及后续上下文追踪。
- 支持网页预览、markdown/text文档、代码片段、diffs/reviews及部署卡片等渲染器类别。
- 允许安全编辑制品以创建新版本，无需直接修改仓库文件。
- 允许用户在消息编辑器中引用制品或制品片段，并将该上下文发送至orchestrator/agent流程。
- 保留现有的差异对比、审查、预览、暂存部署、调度器、目标注册表及护栏行为。
- 将编辑范围限定在制品内容内；代码库写入仍需依赖规划器、计划验证器、调度器及编码代理。

## 能力

### 新能力

- `artifact-preview-editing-workbench`：涵盖制品渲染器元数据、制品版本管理、安全的制品编辑、聊天上下文中的制品引用以及 UI 工作台行为。

### 修改后的能力

- 无。

## 影响

- `apps/api`：制品响应元数据、制品 version/edit API、引用元数据、mission/message 上下文管道、测试。
- `apps/web`：制品工作台 UI、渲染器组件、编辑器控件、版本历史、消息编辑器制品引用、测试。
- 现有的 preview/deployment/diff/review 制品继续可用；未引入新的生产部署提供方、外部市场或任意文件写入路径。
