## 为什么

P18 使 AgentHub 的记忆可审计、可度量，但大多数证据仍是确定性的单元级证明，而非真实工作流证明。需要 P18b 来直接回答产品问题：保存的记忆能否在真实任务中改善 Planner 和编码代理的效果，还是仅仅增加了复杂性？

## 变更内容

- 为可重复的 before/after 或 control/treatment 任务运行添加有界内存有效性演练能力。
- 定义真实任务场景，涵盖用户偏好、项目规则、模式记忆、提示注入拦截和快照一致性。
- 记录每次演练运行的指标基线和处理结果。
- 生成可审计报告，说明内存是有帮助、有损害还是无显著影响。
- 在可行的情况下添加轻量级 CLI/API/test 测试框架，使演练无需手动电子表格工作即可重复。
- 保留 P18 内存语义：内存指导智能体，而目标注册表、计划验证器、护栏和运行时配置仍为执行边界。

## 能力

### 新能力

- `memory-effectiveness-rehearsal`: 对 AgentHub 记忆是否改善规划器和编码代理行为的可重复真实任务评估。

### 修改后的能力

- 无。

## 影响

- 可添加 API 侧的 rehearsal/eval 辅助函数、测试夹具、测试用例及文档。
- 可在 `docs/` 下添加小型报告制品，作为 P18b 冻结证据。
- 可在实现过程中更新 `docs/change-log.md`、`docs/project-state.md` 及 P18b 任务检查清单。
- 不引入新的提供商市场、向量数据库、生产部署或自动长期学习机制。
