## 为什么

AgentHub 已跨越最初的 P0 演示门槛，但项目基线目前混合了 P0 护栏、P1/P2/P4 证据、当前适配器实际情况以及一些过时的验证说明。此变更强化了最终的本地演示，使其能够被展示、解释、重置和冻结，而不会过度宣称其为完整的 IM 多智能体协作平台。

## 变更内容

- 协调基线治理文档，使 `AGENTS.md`、README、`docs/project-state.md` 和 `docs/change-log.md` 能够描述当前本地单用户 Agent 编码工作区的实际情况。
- 修复先前 `agenthub-im-coding-mvp` 变更中已知的 OpenSpec 严格验证问题。
- 仅在确认无关脏文件可丢弃或已有意移入文档后，才进行清理。
- 针对真实 Agent 路径执行并记录浏览器驱动的端到端点击演练：需求 → 计划 → Agent 执行 → 差异对比 → 预览 → 模拟部署。
- 保留并重新验证强制失败的 `ScriptedMockAdapter` 兜底路径。
- 保留并重新验证同一会话中针对窄范围 UI 文本变更的后续路径。
- 添加或记录一个演示重置/干净种子辅助工作流，该工作流能安全备份 SQLite 状态、重置并播种本地演示，同时说明恢复步骤。
- 生成一份适合面试或评委演示的最终演示检查清单和最终项目总结。
- 在验证后冻结最终演示基线。

不打算进行运行时功能扩展。这是一项加固、证据和文档方面的变更。

## 能力

### 新能力

- `demo-baseline-hardening`: 最终 AgentHub 演示加固的治理与证据要求。这是一项文档与验证能力，而非运行时功能扩展。

### 修改后的能力

无。现有运行时能力合约保持不变。工作范围仅限于基础治理、演示验证、重置文档以及最终的解释性文档。

## 影响

- 文档和 OpenSpec 制品：
  - `AGENTS.md`
  - `README.md`
  - `docs/project-state.md`

  - 最终演示检查清单 / 按需提供的总结文档

## 明确非目标

- 完整的即时通讯平台。
- 多用户协作。
- 外部飞书、微信、Slack 或其他即时通讯集成。
- 提供商市场。
- 生产环境部署。
- Docker 沙箱。
- WebSocket 或多玩家传输。
- PR 创建或补丁导出。
- 广泛的任意自然语言编辑。
- 企业审批工作流。
- 移动优先重新设计。
