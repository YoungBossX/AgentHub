## ADDED Requirements
### Requirement: TaskRun 心跳与租约
系统 MUST 记录 TaskRun 存活元数据，以便将本地执行分类为活跃、过期、超时或终止。

#### Scenario: TaskRun 心跳被记录
- **WHEN** 创建或开始执行一个 TaskRun
- **THEN** 系统 MUST 为该 TaskRun 记录 `runnerId`、`lastHeartbeatAt` 和 `leaseExpiresAt` 值
- **并且** 后续的心跳更新 MUST 刷新 `lastHeartbeatAt` 和 `leaseExpiresAt`。

#### Scenario: 正在运行的 TaskRun 变为过期
- **WHEN** TaskRun 处于活跃运行状态
- **且** 其租约已过期且未收到新的心跳
- **THEN** 系统 MUST 将该 TaskRun 标记为过期
- **且** 系统 MUST 未声明适配器已成功完成。

#### Scenario: 过期的 TaskRun 被如实标记
- **WHEN** 过期检测将标记为超时或遗弃的 TaskRun
- **THEN** TaskRun MUST 进入失败、中断、超时或等效的终止状态
- **并且** 审计事件 MUST 记录过期原因、运行器 ID 和租约时间戳。

### Requirement: 过期目标锁清理
系统 MUST 将目标写入锁绑定到 TaskRun 所有者，并仅清理所有者已过期的锁。

#### Scenario: 目标锁记录其所有者
- **WHEN** 写入 TaskRun 会获取目标锁
- **THEN** 锁元数据 MUST 标识 `targetId`、所属 `taskRunId`、
  会话、锁模式、获取时间和租约过期时间。

#### Scenario: 活跃锁未被释放
- **WHEN** 目标锁持有者拥有有效的心跳且租约未过期
- **THEN** 过期锁清理 MUST 不会释放该锁
- **并且** 下游对同一目标的写入任务 MUST 仍被该锁阻塞。

#### Scenario: 过期的所有者锁被释放
- **WHEN** 目标锁的所有者已过期或超时
- **THEN** 清理操作可以释放目标锁
- **并且** 清理操作 MUST 写入一条审计事件，标识该锁、目标 ID、所有者 TaskRun 以及清理原因。

### Requirement: 运行前快照与检查点
系统 MUST 在内置或外部目标上执行写入任务之前，记录一个运行前检查点。

#### Scenario: 写入任务检查点已创建
- **WHEN** 一个写入型 TaskRun 即将开始
- **THEN** 系统 MUST 记录目标 ID、目标根目录、允许路径、禁止路径、基准提交、Git 状态、脏文件、计划文件以及检查点创建时间。

#### Scenario: 外部目标检查点已创建
- **WHEN** 写入 TaskRun 的目标是一个已注册的外部项目
- **THEN** 检查点 MUST 使用目标注册表中的外部目标根目录和路径策略
- **并且** 检查点 MUST 不暴露被禁止的路径内容。

#### Scenario: 检查点元数据在恢复时可见
- **WHEN** TaskRun 有一个运行前检查点
- **THEN** 重试与恢复 API MUST 能够读取检查点元数据
- **并且** UI 或 API MUST 暴露足够的元数据以解释重试安全性。

### Requirement: 重试幂等性
系统 MUST 根据先前的运行状态、检查点元数据和当前目标的清洁度做出重试决策。

#### Scenario: 重试记录上一次运行
- **WHEN** 用户重试失败或中断的 TaskRun
- **THEN** 重试的 TaskRun MUST 记录前一次运行的 ID、失败摘要、重试模式以及可用的检查点引用。

#### Scenario: 不安全的重试被阻止
- **WHEN** 目标工作树在检查点或计划的安全路径之外存在脏文件
- **THEN** 自动重试 MUST 被阻止
- **并且** 系统 MUST 说明需要显式的恢复决策。

#### Scenario: 安全重试继续
- **WHEN** 目标工作树符合检查点安全条件
- **且** 依赖与目标锁定规则允许执行
- **THEN** 重试可通过现有 TaskRun 路径创建新的 TaskRun
- **且** 重试历史 MUST 保持可追溯。

### Requirement: 故障传播加固
系统 MUST 在前提条件失败后阻止下游执行、预览和模拟部署。

#### Scenario: 失败的依赖项阻塞下游
- **WHEN** 上游依赖项失败、被中断、超时或被标记为过期
- **THEN** 下游任务 MUST 保持阻塞状态
- **且** 其调度器状态 MUST 标识出失败的前置条件。

#### Scenario: 成功后的重试解除下游阻塞
- **WHEN** 失败的依赖项被重试或成功恢复
- **THEN** 调度器 MUST 重新评估下游任务
- **并且** 仅当依赖项、锁、审批和冲突检查通过时，下游任务才可能变为可运行状态。

#### Scenario: 预览和部署受控
- **WHEN** 编码前置条件失败或被阻止
- **THEN** 预览创建 MUST 不会自动运行
- **并且** 模拟部署 MUST 不会自动运行。

### Requirement: 冲突检测
系统 MUST 会检测常见的写入冲突，并在遇到复杂变更时停止，而非自动合并。

#### Scenario: 检测到文件重叠冲突
- **WHEN** 同一会话中的两个写入任务的目标计划文件存在重叠
- **且** 它们可能并发执行或按不安全顺序运行
- **THEN** 调度器 MUST 识别出文件重叠冲突
- **且** 至少一个任务 MUST 等待、阻塞或需要显式恢复。

#### Scenario: 检测到脏工作区冲突
- **WHEN** 目标工作树包含未被检查点、所选制品或计划安全路径覆盖的脏文件
- **THEN** 调度器 MUST 识别到脏工作树冲突
- **且** 系统 MUST 不会自动启动不安全的写入 TaskRun。

#### Scenario: 检测到合约漂移冲突
- **WHEN** 下游任务引用的应用合约版本、哈希或 ID 与当前活跃合约不再匹配
- **THEN** 调度器 MUST 识别出合约漂移冲突
- **且** 下游实现 MUST 不得在过时合约上下文中静默继续执行。

#### Scenario: 复杂冲突无法自动合并
- **WHEN** 冲突检测发现文件变更重叠、工作区脏状态或契约漂移
- **THEN** 系统在 P10 阶段 **不得** 自动合并冲突
- **且** 恢复选项 **必须** 明确且可审计。

### Requirement: 恢复操作
系统 MUST 为过期运行、过期锁、重试决策以及下游流水线控制提供明确的可审计恢复操作。

#### Scenario: 将过期任务标记为失败
- **WHEN** 用户或调度器的恢复操作将过期的 TaskRun 标记为失败
- **THEN** TaskRun MUST 变为终态
- **并且** 审计事件 MUST 记录操作者、原因、先前状态和新状态。

#### Scenario: 释放过期锁
- **WHEN** 恢复操作会释放一个过期的目标锁
- **THEN** 仅当锁所属的 TaskRun 已过期或处于终止状态时，该锁 MUST 被释放
- **并且** 审计事件 MUST 记录被释放的锁及其所有者。

#### Scenario: 从当前状态重试
- **WHEN** 用户选择从当前状态重试
- **THEN** 系统 MUST 检查脏工作区和冲突条件
- **并且** 重试 MUST 被阻止或通过可追踪的恢复事件执行。

#### Scenario: 从检查点重试
- **WHEN** 用户从检查点选择重试
- **THEN** 系统 MUST 验证检查点恢复是否安全
- **并且** 系统 MUST 不会破坏性地重置允许目标路径之外的文件。

#### Scenario: 停止或恢复下游流水线
- **WHEN** 用户停止或恢复下游管道的推进
- **THEN** 调度器 MUST 显式更新下游资格
- **并且** 该操作 MUST 生成审计事件。

### Requirement: P10 基线保留
系统 MUST 在增加调度器健壮性和冲突恢复能力的同时，保留 P6/P7/P8/P9 行为。

#### Scenario: 执行 P10 冻结评审
- **WHEN** P10 已审查完毕，准备冻结
- **THEN** 需演练过期任务、过期锁、失败依赖、重试及冲突场景 MUST
- **且** P9 外部项目工作区模式 MUST 需保持可运行
- **且** P8 依赖与目标锁语义 MUST 需保持完整
- **且** 不得声称存在未经验证的 Claude/Codex 成功 MUST。

#### Scenario: 请求了超出范围的能力
- **WHEN** 请求需要分布式工作节点、Docker 沙箱、生产环境部署、提供商市场、PR 创建、自动 Git 冲突合并、多用户即时通讯或企业级 RBAC
- **THEN** P10 MUST 如实拒绝或推迟该请求
- **并且** 不得 MUST 在已注册的目标边界之外静默执行。
