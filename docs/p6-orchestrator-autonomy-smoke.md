## Caveats

- This was an API-driven smoke using FastAPI `TestClient`, not a browser click
  rehearsal.
- The preview was reported healthy while the smoke script was running. A
  follow-up `curl` after the one-shot TestClient process exited could not reach
  the preview URL, so long-lived preview availability should be verified again
  under `pnpm dev:api` during a later browser rehearsal.
- The Review artifact was generated by the deterministic scripted review path,
  not by real Claude/Codex review.
- ScriptedMock fallback was not needed because the real Claude Code run
  completed successfully.
- The smoke did not implement P6-2 context pack, a backend target, full-stack
  app generation, production deploy, provider marketplace, or multi-user IM.
